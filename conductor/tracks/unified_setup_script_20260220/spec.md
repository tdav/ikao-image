# Track Specification: Unified Setup and Deployment Script

## Overview
Этот трек направлен на обновление скрипта setup.sh для автоматизации полной сборки и подготовки к развертыванию всех компонентов проекта IKAO-OFIQ: нативного моста (C++), консольного приложения и REST API сервиса.

## Functional Requirements
Скрипт должен выполнять следующие действия:
1.  **Очистка:** Удаление старой директории dist/ перед началом сборки.
2.  **Сборка C++:** Компиляция ofiq_bridge.cpp в libofiq_bridge.so с линковкой оригинальной библиотеки.
3.  **Публикация .NET:**
    - Публикация проекта консольного приложения в dist/console.
    - Публикация проекта REST API в dist/api.
4.  **Развертывание библиотек:**
    - Копирование libofiq_lib.so, libonnxruntime.so.1.18.1 и libofiq_bridge.so в подпапки приложений.
    - Создание символической ссылки libonnxruntime.so.
5.  **Развертывание ресурсов:**
    - Копирование моделей и конфигурационных файлов JAXN в единую директорию dist/config.

## Non-Functional Requirements
- **Надежность:** Использование set -e для мгновенной остановки при любой ошибке.
- **Информативность:** Вывод четких сообщений о каждом этапе сборки.
- **Портативность:** Скрипт предназначен исключительно для Linux (x64).

## Acceptance Criteria
- После запуска ./setup.sh в папке dist/ создается готовая к работе структура.
- Консольное приложение и API запускаются из своих папок в dist/ без дополнительных настроек путей.
- Все необходимые зависимости (.so) присутствуют в выходных папках.
